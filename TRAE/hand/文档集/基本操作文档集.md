# 基本操作文档集

## 异常处理规范

### 异常统一使用MtException抛出

#### 1. 目的
- 统一异常处理机制，确保系统异常能够被正确捕获和处理
- 提供一致的错误信息格式，方便前端和其他服务处理
- 便于日志记录和监控

#### 2. 使用方法
- 在服务层和业务逻辑中，当遇到异常情况时，使用MtException抛出
- 确保异常信息清晰明了，包含必要的错误码和错误描述
- 避免直接抛出原始异常，应转换为MtException

#### 3. 示例代码
```java
if (condition) {
    throw new MtException("error.code.null");
}
```

#### 4. 导入包
```java
import io.tarzan.core.sys.MtException;
```

#### 5. 注意事项
- 异常信息应包含足够的上下文信息，便于排查问题
- 对于外部服务调用异常，应捕获并转换为MtException
- 确保异常处理不会影响系统的正常运行

#### 6. 错误消息获取

##### 6.1 使用MtCommonRpcClient获取错误消息

**类路径**：`org.tarzan.boot.common.client.MtCommonRpcClient`

**主要方法**：
- `getErrorMessage` - 获取单个错误信息
- `getBatchErrorMessage` - 批量获取错误信息

**示例代码**：
```java
import org.tarzan.boot.common.client.MtCommonRpcClient;

String errorMessage = mtCommonRpcClient.getErrorMessage(tenantId, "ERROR_CODE", "param1", "param2");
System.out.println("错误信息：" + errorMessage);
```

#### 7. 最佳实践
- 定义统一的错误码体系
- 异常信息应国际化，支持多语言
- 对异常进行分类，便于不同级别的处理
- 使用MtCommonRpcClient获取错误消息，确保错误信息的一致性
- 批量获取错误消息以提高性能

## MtCustomDbRepository通用数据库操作类

### 1. 概述

**类路径**：`io.tarzan.core.repository.MtCustomDbRepository`

**作用**：提供通用的实体类数据库批量操作功能，适用于任何实体类的批量增删改查操作。

**主要特点**：
- 支持批量操作，提高性能
- 支持事务管理
- 支持主键生成
- 适用于任何实体类

### 2. 使用方法

#### 2.1 注入依赖

```java
import io.tarzan.core.repository.MtCustomDbRepository;

@Autowired
private MtCustomDbRepository mtCustomDbRepository;
```

#### 2.2 批量插入方法

**批量插入**：
```java
// 基本批量插入
mtCustomDbRepository.batchInsert(entityList);

// 指定批次大小的批量插入
mtCustomDbRepository.batchInsert(entityList, 1000);
```

**带主键的批量插入**：
```java
// 带主键的批量插入
mtCustomDbRepository.batchInsertWithPrimaryKey(entityList);

// 指定批次大小的带主键批量插入
mtCustomDbRepository.batchInsertWithPrimaryKey(entityList, 1000);
```

#### 2.3 批量更新方法

**全量更新**：
```java
// 基本批量更新
mtCustomDbRepository.batchUpdate(entityList);

// 指定批次大小的批量更新
mtCustomDbRepository.batchUpdate(entityList, 1000);
```

**选择性更新**：
```java
// 选择性批量更新（只更新非空字段）
mtCustomDbRepository.batchUpdateSelective(entityList);

// 指定批次大小的选择性批量更新
mtCustomDbRepository.batchUpdateSelective(entityList, 1000);
```

#### 2.4 批量删除方法

```java
// 根据ID列表批量删除
mtCustomDbRepository.batchDelete(idList, EntityClass.class);

// 指定批次大小的批量删除
mtCustomDbRepository.batchDelete(idList, EntityClass.class, 1000);
```

#### 2.5 主键生成方法

**单个主键**：
```java
// 获取下一个主键值
Long primaryKey = mtCustomDbRepository.getNextPrimaryKey(EntityClass.class);
```

**多个主键**：
```java
// 获取多个主键值
List<Long> primaryKeys = mtCustomDbRepository.getNextPrimaryKeys(EntityClass.class, 10);
```

**CID生成**：
```java
// 获取下一个CID值列表
List<Long> cids = mtCustomDbRepository.getNextCids(EntityClass.class, 10);
```

### 3. 使用场景

#### 3.1 批量导入数据

```java
// 批量插入数据
mtCustomDbRepository.batchInsert(importDataList, 1000);
```

#### 3.2 批量更新记录

```java
// 批量更新数据（选择性更新）
mtCustomDbRepository.batchUpdateSelective(updateList, 1000);
```

#### 3.3 批量删除数据

```java
// 批量删除数据
mtCustomDbRepository.batchDelete(deleteIds, EntityClass.class, 1000);
```

#### 3.4 预先生成主键

```java
// 预先生成主键
List<Long> primaryKeys = mtCustomDbRepository.getNextPrimaryKeys(EntityClass.class, list.size());
// 设置主键到实体对象
for (int i = 0; i < list.size(); i++) {
    list.get(i).setId(primaryKeys.get(i));
}
// 批量插入带主键的数据
mtCustomDbRepository.batchInsertWithPrimaryKey(list);
```

### 4. 最佳实践

- **合理设置批次大小**：根据数据量大小设置合适的批次大小，一般建议500-2000
- **使用事务管理**：在Service层添加`@Transactional`注解确保数据一致性
- **优先使用选择性更新**：对于部分字段更新场景，使用`batchUpdateSelective`提高性能
- **预先生成主键**：对于需要在插入前知道主键的场景，使用主键生成方法
- **异常处理**：捕获并转换异常为`MtException`，确保错误信息清晰

## Long字段判空方法

### 1. 概述

**类路径**：`io.tarzan.core.util.MtIdHelper`

**作用**：提供Long类型字段的判空方法，用于判断ID类型字段是否为空。

**主要特点**：
- 提供标准化的Long字段判空方法
- 支持Stream流操作中的过滤操作
- 统一的判空逻辑，避免空指针异常

### 2. 主要方法

#### 2.1 isIdNotNull方法

**功能**：判断Long类型字段是否不为空且大于0

**方法签名**：
```java
public static boolean isIdNotNull(Long id)
```

**返回值**：
- `true` - 字段不为空且大于0
- `false` - 字段为空或小于等于0

#### 2.2 isIdNull方法

**功能**：判断Long类型字段是否为空或小于等于0

**方法签名**：
```java
public static boolean isIdNull(Long id)
```

**返回值**：
- `true` - 字段为空或小于等于0
- `false` - 字段不为空且大于0

### 3. 使用方法

#### 3.1 导入包

```java
import io.tarzan.core.util.MtIdHelper;
```

#### 3.2 基本判空使用

```java
// 判断ID不为空
if (MtIdHelper.isIdNotNull(entityId)) {
    // 处理非空逻辑
}

// 判断ID为空
if (MtIdHelper.isIdNull(entityId)) {
    // 处理空逻辑
}
```

#### 3.3 Stream流中使用

```java
// 在Stream流中过滤非空ID
List<Long> validIds = entityList.stream()
        .map(Entity::getId)
        .filter(MtIdHelper::isIdNotNull)
        .distinct()
        .collect(Collectors.toList());
```

### 4. 最佳实践

- **统一使用MtIdHelper**：所有Long类型的ID字段判空都应使用MtIdHelper方法
- **结合Stream流使用**：在处理集合数据时，使用filter(MtIdHelper::isIdNotNull)进行过滤
- **避免直接判空**：不使用`id != null && id > 0`的方式，统一使用工具类方法
- **批量处理优化**：先过滤出非空ID，再进行批量查询，减少不必要的RPC调用
- **空值检查**：在调用RPC方法前，使用CollectionUtils.isNotEmpty检查ID列表是否为空

## @LovValue注解用法

### 1. 概述

**类路径**：`org.hzero.boot.platform.lov.annotation.LovValue`

**作用**：将代码值自动转换为可读的显示文本，实现代码值与显示文本的分离。

**主要特点**：
- 自动将代码值转换为显示文本
- 支持国际化配置
- 减少硬编码，提高代码可维护性
- 适用于DTO类的字段注解

### 2. 注解参数

#### 2.1 value参数

**功能**：指定值集代码（Value Set Code）

**示例**：`"MT.YES_NO"`

**说明**：值集代码对应系统中配置的值集定义，用于获取代码对应的显示文本

#### 2.2 meaningField参数

**功能**：指定存储显示文本的字段名称

**示例**：`"calcFlagMeaning"`

**说明**：框架会自动将代码值转换为显示文本，并填充到这个指定的字段中

### 3. 使用方法

#### 3.1 导入包

```java
import org.hzero.boot.platform.lov.annotation.LovValue;
```

#### 3.2 基本使用示例

```java
@ApiModelProperty("产能计算点")
@ExcelColumn(zh = "产能计算点", en = "Calc Flag", order = 5)
@LovValue(value = "MT.YES_NO", meaningField = "calcFlagMeaning")
private String calcFlag;

private String calcFlagMeaning;
```

#### 3.3 完整DTO类示例

```java
import org.hzero.boot.platform.lov.annotation.LovValue;

public class HpsAreaCapacityResponseDTO implements Serializable {
    // ...其他字段
    
    @ApiModelProperty("产能计算点")
    @LovValue(value = "MT.YES_NO", meaningField = "calcFlagMeaning")
    private String calcFlag;
    
    private String calcFlagMeaning;
    
    // getter和setter方法
    public String getCalcFlag() {
        return calcFlag;
    }
    
    public void setCalcFlag(String calcFlag) {
        this.calcFlag = calcFlag;
    }
    
    public String getCalcFlagMeaning() {
        return calcFlagMeaning;
    }
    
    public void setCalcFlagMeaning(String calcFlagMeaning) {
        this.calcFlagMeaning = calcFlagMeaning;
    }
}
```

### 4. 工作原理

1. **代码值存储**：`calcFlag`字段存储原始代码值（如 "Y"/"N"）
2. **自动转换**：框架在序列化响应时，自动根据值集代码查询对应的显示文本
3. **填充显示文本**：将查询到的显示文本填充到`calcFlagMeaning`字段中
4. **前端展示**：前端可以直接使用`calcFlagMeaning`显示友好的文本（如 "是"/"否"）

### 5. 使用场景

#### 5.1 状态字段显示

```java
@LovValue(value = "MT.YES_NO", meaningField = "statusMeaning")
private String status;
private String statusMeaning;
```

#### 5.2 类型字段显示

```java
@LovValue(value = "MT.MATERIAL_TYPE", meaningField = "materialTypeMeaning")
private String materialType;
private String materialTypeMeaning;
```

#### 5.3 Excel导出显示

结合@ExcelColumn注解使用，在Excel导出时显示友好文本：

```java
@ExcelColumn(zh = "状态", en = "Status", order = 1)
@LovValue(value = "MT.STATUS", meaningField = "statusMeaning")
private String status;
private String statusMeaning;
```

### 6. 配置要求

1. **导入依赖**：确保项目中引入了HZero boot依赖
2. **定义含义字段**：在类中必须定义对应的含义字段（如`calcFlagMeaning`）
3. **值集配置**：在系统中配置对应的值集（如"MT.YES_NO"）

### 7. 最佳实践

- **代码解耦**：业务代码只需处理代码值，显示文本由框架自动处理
- **国际化支持**：值集支持多语言配置，前端可根据语言环境显示对应文本
- **减少硬编码**：避免在代码中直接写死显示文本
- **易于维护**：值集配置变更时，无需修改代码
- **命名规范**：含义字段命名为原字段名+Meaning（如`calcFlag`对应`calcFlagMeaning`）
