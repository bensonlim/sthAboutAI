# 基本操作文档集

## 异常处理规范

### 异常统一使用MtException抛出

#### 1. 目的
- 统一异常处理机制，确保系统异常能够被正确捕获和处理
- 提供一致的错误信息格式，方便前端和其他服务处理
- 便于日志记录和监控

#### 2. 使用方法
- 在服务层和业务逻辑中，当遇到异常情况时，使用MtException抛出
- 确保异常信息清晰明了，包含必要的错误码和错误描述
- 避免直接抛出原始异常，应转换为MtException

#### 3. 示例代码
```java
if (condition) {
    throw new MtException("error.code.null");
}
```

#### 4. 导入包
```java
import io.tarzan.core.sys.MtException;
```

#### 5. 注意事项
- 异常信息应包含足够的上下文信息，便于排查问题
- 对于外部服务调用异常，应捕获并转换为MtException
- 确保异常处理不会影响系统的正常运行

#### 6. 错误消息获取

##### 6.1 使用MtCommonRpcClient获取错误消息

**类路径**：`org.tarzan.boot.common.client.MtCommonRpcClient`

**主要方法**：
- `getErrorMessage` - 获取单个错误信息
- `getBatchErrorMessage` - 批量获取错误信息

**示例代码**：
```java
import org.tarzan.boot.common.client.MtCommonRpcClient;

String errorMessage = mtCommonRpcClient.getErrorMessage(tenantId, "ERROR_CODE", "param1", "param2");
System.out.println("错误信息：" + errorMessage);
```

#### 7. 最佳实践
- 定义统一的错误码体系
- 异常信息应国际化，支持多语言
- 对异常进行分类，便于不同级别的处理
- 使用MtCommonRpcClient获取错误消息，确保错误信息的一致性
- 批量获取错误消息以提高性能

## MtCustomDbRepository通用数据库操作类

### 1. 概述

**类路径**：`io.tarzan.core.repository.MtCustomDbRepository`

**作用**：提供通用的实体类数据库批量操作功能，适用于任何实体类的批量增删改查操作。

**主要特点**：
- 支持批量操作，提高性能
- 支持事务管理
- 支持主键生成
- 适用于任何实体类

### 2. 使用方法

#### 2.1 注入依赖

```java
import io.tarzan.core.repository.MtCustomDbRepository;

@Autowired
private MtCustomDbRepository mtCustomDbRepository;
```

#### 2.2 批量插入方法

**批量插入**：
```java
// 基本批量插入
mtCustomDbRepository.batchInsert(entityList);

// 指定批次大小的批量插入
mtCustomDbRepository.batchInsert(entityList, 1000);
```

**带主键的批量插入**：
```java
// 带主键的批量插入
mtCustomDbRepository.batchInsertWithPrimaryKey(entityList);

// 指定批次大小的带主键批量插入
mtCustomDbRepository.batchInsertWithPrimaryKey(entityList, 1000);
```

#### 2.3 批量更新方法

**全量更新**：
```java
// 基本批量更新
mtCustomDbRepository.batchUpdate(entityList);

// 指定批次大小的批量更新
mtCustomDbRepository.batchUpdate(entityList, 1000);
```

**选择性更新**：
```java
// 选择性批量更新（只更新非空字段）
mtCustomDbRepository.batchUpdateSelective(entityList);

// 指定批次大小的选择性批量更新
mtCustomDbRepository.batchUpdateSelective(entityList, 1000);
```

#### 2.4 批量删除方法

```java
// 根据ID列表批量删除
mtCustomDbRepository.batchDelete(idList, EntityClass.class);

// 指定批次大小的批量删除
mtCustomDbRepository.batchDelete(idList, EntityClass.class, 1000);
```

#### 2.5 主键生成方法

**单个主键**：
```java
// 获取下一个主键值
Long primaryKey = mtCustomDbRepository.getNextPrimaryKey(EntityClass.class);
```

**多个主键**：
```java
// 获取多个主键值
List<Long> primaryKeys = mtCustomDbRepository.getNextPrimaryKeys(EntityClass.class, 10);
```

**CID生成**：
```java
// 获取下一个CID值列表
List<Long> cids = mtCustomDbRepository.getNextCids(EntityClass.class, 10);
```

### 3. 使用场景

#### 3.1 批量导入数据

```java
// 批量插入数据
mtCustomDbRepository.batchInsert(importDataList, 1000);
```

#### 3.2 批量更新记录

```java
// 批量更新数据（选择性更新）
mtCustomDbRepository.batchUpdateSelective(updateList, 1000);
```

#### 3.3 批量删除数据

```java
// 批量删除数据
mtCustomDbRepository.batchDelete(deleteIds, EntityClass.class, 1000);
```

#### 3.4 预先生成主键

```java
// 预先生成主键
List<Long> primaryKeys = mtCustomDbRepository.getNextPrimaryKeys(EntityClass.class, list.size());
// 设置主键到实体对象
for (int i = 0; i < list.size(); i++) {
    list.get(i).setId(primaryKeys.get(i));
}
// 批量插入带主键的数据
mtCustomDbRepository.batchInsertWithPrimaryKey(list);
```

### 4. 最佳实践

- **合理设置批次大小**：根据数据量大小设置合适的批次大小，一般建议500-2000
- **使用事务管理**：在Service层添加`@Transactional`注解确保数据一致性
- **优先使用选择性更新**：对于部分字段更新场景，使用`batchUpdateSelective`提高性能
- **预先生成主键**：对于需要在插入前知道主键的场景，使用主键生成方法
- **异常处理**：捕获并转换异常为`MtException`，确保错误信息清晰
