# 扩展属性处理

## 1. 概述

扩展属性是一种在不修改主表结构的情况下，为实体添加额外属性的机制。在 APS 系统中，通过 `MtExtendRepository` 接口来处理扩展属性的查询和管理。本文档将详细介绍如何使用 `MtExtendRepository` 进行扩展属性的处理。

## 2. MtExtendRepository 接口

`MtExtendRepository` 是一个用于处理扩展表属性的仓库接口，它提供了对实体扩展属性的查询和管理功能。

### 2.1 核心方法

| 方法名 | 功能 | 参数 | 返回值 |
|-------|------|------|-------|
| `tableLimitAttrNameQuery` | 查询指定实体类的扩展表属性定义 | tenantId: 租户ID<br>entityClass: 实体类 Class 对象 | MtExtendRpcVO: 包含扩展表配置和扩展列信息的对象 |
| `attrPropertyQuery` | 根据条件查询扩展属性值 | tenantId: 租户ID<br>entityClass: 实体类 Class 对象<br>attrQueryVO: 查询条件对象 | List<MtAttrVO>: 扩展属性值列表 |

## 3. 使用步骤

### 3.1 注入 MtExtendRepository

首先需要在服务或仓库类中注入 `MtExtendRepository` 实例：

```java
@Autowired
private MtExtendRepository mtExtendRepository;
```

### 3.2 查询扩展属性定义

使用 `tableLimitAttrNameQuery` 方法查询实体类的扩展属性定义：

1. 调用方法获取扩展表配置
2. 检查返回结果是否不为空
3. 从返回结果中提取扩展列信息
4. 可以根据需要对扩展列进行过滤（如只获取启用的属性）
5. 可以根据需要对扩展列进行排序（如按序列排序）
6. 提取需要的属性信息（如属性含义）

**代码示例**：

```java
// 扩展表的属性名
MtExtendRpcVO mtExtendRpcVO = mtExtendRepository.tableLimitAttrNameQuery(tenantId, MtMaterialLot.class);
if (mtExtendRpcVO != null && CollectionUtils.isNotEmpty(mtExtendRpcVO.getExtendColumnList())) {
    List<String> attrs = mtExtendRpcVO.getExtendColumnList().stream()
            .filter(t -> YES.equalsIgnoreCase(t.getEnableFlag()))
            .sorted(Comparator.comparing(MtExtendColumnRpcVO::getSequence))
            .map(MtExtendColumnRpcVO::getAttrMeaning).collect(Collectors.toList());
    // 后续处理...
}
```

### 3.3 查询扩展属性值

使用 `attrPropertyQuery` 方法查询实体实例的扩展属性值：

1. 创建 `MtAttrQueryVO` 对象
2. 设置查询条件（如实体ID和属性名称）
3. 调用方法获取扩展属性值
4. 处理返回的扩展属性值

**代码示例**：

```java
// 根据输入参数获取拓展属性
MtAttrQueryVO extendVO = new MtAttrQueryVO();
extendVO.setKeyId(dto.getMaterialLotId());
extendVO.setAttrName(dto.getAttrName());
return mtExtendRepository.attrPropertyQuery(tenantId, MtMaterialLot.class, extendVO);
```

## 4. 实际应用场景

### 4.1 Excel 模板生成

在 `MtMaterialLotServiceImpl` 类中，使用 `tableLimitAttrNameQuery` 方法获取物料批的扩展属性名称，用于在 Excel 模板中添加对应的列：

```java
// 扩展表的属性名
MtExtendRpcVO mtExtendRpcVO = mtExtendRepository.tableLimitAttrNameQuery(tenantId, MtMaterialLot.class);
if (mtExtendRpcVO != null && CollectionUtils.isNotEmpty(mtExtendRpcVO.getExtendColumnList())) {
    List<String> attrs = mtExtendRpcVO.getExtendColumnList().stream()
            .filter(t -> YES.equalsIgnoreCase(t.getEnableFlag()))
            .sorted(Comparator.comparing(MtExtendColumnRpcVO::getSequence))
            .map(MtExtendColumnRpcVO::getAttrMeaning).collect(Collectors.toList());
    // 将扩展属性添加到 Excel 模板中
    attrs.forEach(attr -> cellValueList.add(new MtMaterialLotTempExcelVO(attr, cellStyle,
            this.createCellComment(p, cellValueList.size() + 1, rowIndex, finalLovValueMap
                    .get(MtMesConstants.INVENTORY_IMPORT_MODEL_COLUMN.COLUMN_27)))));
}
```

### 4.2 扩展属性查询

在 `MtMaterialLotRepositoryImpl` 类中，使用 `attrPropertyQuery` 方法根据物料批 ID 和属性名称查询扩展属性值：

```java
@Override
public List<MtAttrVO> materialLotLimitAttrQuery(Long tenantId, MtMaterialLotAttrVO2 dto) {
    if (dto == null || MtIdHelper.isIdNull(dto.getMaterialLotId())) {
        throw new MtException("MT_MATERIAL_LOT_0001", mtCommonRpcClient.getErrorMessage(tenantId,
                "MT_MATERIAL_LOT_0001", "materialLotId", "【API:materialLotLimitAttrQuery】"));
    }

    // 根据输入参数获取拓展属性
    MtAttrQueryVO extendVO = new MtAttrQueryVO();
    extendVO.setKeyId(dto.getMaterialLotId());
    extendVO.setAttrName(dto.getAttrName());
    return mtExtendRepository.attrPropertyQuery(tenantId, MtMaterialLot.class, extendVO);
}
```

## 5. 扩展属性数据结构

### 5.1 MtExtendRpcVO

`MtExtendRpcVO` 是扩展表配置的核心数据结构，包含以下主要字段：

- `enableFlag`：扩展表是否启用
- `mainTable`：主表名称
- `mainTableKey`：主表主键
- `attrType`：扩展属性类型（ROW 或 COLUMN）
- `extendColumnList`：扩展列列表

### 5.2 MtExtendColumnRpcVO

`MtExtendColumnRpcVO` 是扩展列的核心数据结构，包含以下主要字段：

- `attrName`：属性名称
- `attrMeaning`：属性含义
- `enableFlag`：是否启用
- `sequence`：排序序列

### 5.3 MtAttrQueryVO

`MtAttrQueryVO` 是扩展属性查询的条件对象，包含以下主要字段：

- `keyId`：实体ID
- `attrName`：属性名称

### 5.4 MtAttrVO

`MtAttrVO` 是扩展属性值的返回对象，包含以下主要字段：

- `attrName`：属性名称
- `attrValue`：属性值

## 6. 最佳实践

1. **注入方式**：使用 `@Autowired` 注解注入 `MtExtendRepository` 实例
2. **参数验证**：在调用方法前，验证必要的参数（如 tenantId、entityClass 等）
3. **结果处理**：对返回结果进行非空检查，避免空指针异常
4. **流式处理**：使用 Java 8 流式 API 对扩展列进行过滤、排序和转换
5. **异常处理**：合理处理可能出现的异常，提供清晰的错误信息
6. **性能优化**：对于批量操作，考虑使用批量查询方法，减少数据库交互次数

## 7. 总结

`MtExtendRepository` 提供了以下核心功能：

1. **查询扩展属性定义**：通过 `tableLimitAttrNameQuery` 方法获取实体类的扩展属性定义，包括属性名称、含义、序列等信息。
2. **查询扩展属性值**：通过 `attrPropertyQuery` 方法根据实体ID和属性名称查询扩展属性值。

使用 `MtExtendRepository` 可以方便地处理实体的扩展属性，而不需要直接编写 SQL 语句，提高了代码的可维护性和可读性。

在实际应用中，`MtExtendRepository` 常用于以下场景：
- 生成包含扩展属性的 Excel 模板
- 查询实体的扩展属性值
- 验证扩展属性的有效性
- 处理扩展属性的批量操作

通过合理使用 `MtExtendRepository`，可以使代码更加简洁、高效，同时也能更好地满足业务需求的变化。
