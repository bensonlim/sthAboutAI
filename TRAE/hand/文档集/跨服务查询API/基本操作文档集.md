# 基本操作文档集

## 异常处理规范

### 异常统一使用MtException抛出

#### 1. 目的
- 统一异常处理机制，确保系统异常能够被正确捕获和处理
- 提供一致的错误信息格式，方便前端和其他服务处理
- 便于日志记录和监控

#### 2. 使用方法
- 在服务层和业务逻辑中，当遇到异常情况时，使用MtException抛出
- 确保异常信息清晰明了，包含必要的错误码和错误描述
- 避免直接抛出原始异常，应转换为MtException

#### 3. 示例代码
```java
if (condition) {
    throw new MtException("error.code.null");
}
```

#### 4. 导入包
```java
import io.tarzan.core.sys.MtException;
```

#### 5. 注意事项
- 异常信息应包含足够的上下文信息，便于排查问题
- 对于外部服务调用异常，应捕获并转换为MtException
- 确保异常处理不会影响系统的正常运行

#### 6. 错误消息获取

##### 6.1 使用MtCommonRpcClient获取错误消息

**类路径**：`org.tarzan.boot.common.client.MtCommonRpcClient`

**主要方法**：
- `getErrorMessage` - 获取单个错误信息
- `getBatchErrorMessage` - 批量获取错误信息

**示例代码**：
```java
import org.tarzan.boot.common.client.MtCommonRpcClient;

String errorMessage = mtCommonRpcClient.getErrorMessage(tenantId, "ERROR_CODE", "param1", "param2");
System.out.println("错误信息：" + errorMessage);
```

#### 7. 最佳实践
- 定义统一的错误码体系
- 异常信息应国际化，支持多语言
- 对异常进行分类，便于不同级别的处理
- 使用MtCommonRpcClient获取错误消息，确保错误信息的一致性
- 批量获取错误消息以提高性能
